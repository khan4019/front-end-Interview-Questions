Closure

Question: What is closure? How closure creates encapsulation?

Answer: If you have a function within a function, execution of the inner function will create a scope inside of the outer function- a nested scope. Because the inside function scope is enclosed by the outer function scope, the inner function scope is called a closure (I invented this definition). Remember, to be a closure, you dont have to return a function, you just have to call a function inside a function.


function counter(){
    var i = 0;    
    return function(){
       return i++;
    }
}
               

var a = counter();
a(); //0
a(); //1
a(); //2
a(); //3
a(); //4

Encapsulation: You would not be able to access (get or set) the value of "i" from outside. Hence the privacy of the value of "i" is maintained secretly. However, you execute a(), the value of "i" will be increased by one. This is called encapsulation in object oriented programming.

string to integer

Question: What is the best way to convert string to number in JavaScript?

Answer: There are nine different ways to convert a string to number

  1. Use the Number constructor and pass the number
  2. use parseInt
  3. use unary plus (+)
  4. use parseInt with a base
  5. use bitwise right shift
  6. multiply by one will cause force cast
  7. bitwise NOT
  8. bitwise OR
  9. just deduct 0 from the string (number in string type)

Number("13268"); //constructor

parseInt("13268"); //parseInt

+"13268"; //unary +

parseInt("13268", 10); //parseInt with base

"13268" >> 0; //bitwise >> (right shift)
"13268" * 1; //multiply by 1
~~"13268"; // bitwise NOT
"13268" | 0 //bitwise OR
"13268" - 0; // minus 0

for in looping

Question: If you loop through [10, 20, 30] by using for-in loop and want to get the sum of all the elements, how will you do it?


var numbers = [10, 20, 30];
var sum = 0;
for(var num in numbers){
  sum += num; 
}

console.log(sum);

Answer: 0012

Basic: for in always loop over keys. JavaScript array is inherited from object and all the elements in the array is like key value pair in the object{0:10, 1:20, 2:30} where key is 0, 1, 2 etc. When you are accessing values by passing index, you are actully accessing by key.

thats why when an index doesnt exist, Javascript returns undefined not array out of index. As the index(key) doesnt exists in the object.

Explanation: as for in looping over keys and keys are always string type, its concatenating 0+'0' which becomes '00' and then u add '1' and '2'. Which eventually results '0012'. Quite unexpected, isnt it?

solution: the solution of this problem is to use num as index to access the value or use a simple for loop


var numbers = [10, 20, 30];
var sum = 0;
for(var num in numbers){
  sum += numbers[num]; 
}

console.log(sum);//60

responsive

Question: What are the things you would consider to make a website responsive?

Answer: It depends on what I am trying to do but you can talk about a some of them-

  • layout, keep it simple. dont use div, absolute position or float unnecessarily. kill inline styles. use HTML5 doctype, reset stylesheet, semantic core layout.
  • media queries, to load stylesheet based on the size of the media or apply rules based on the orientation of the media
  • breakpoints, choose the right breakpoints to switch style. common breakpoints are <480px, <768px, >768px. There are three nice to have breakpoints <320px, >769px and <1024px, >1024px.
  • flexible, don't use fixed width rather use relative to viewport size. For example, 50% of the viewport will always be half of the screen and doesn't depend on the device. This is also important for font-size and it is recommended to use something relative like em
  • pictures, pictures are a big of a deal for responsive websites. Right now you can change src attribute of an image based on the media query alone. Hence you can use background-image or use different picture versions and show/hide their parent in media queries.
  • max-*, min-*, you can use max-width:768px; min-height:350px; to apply style
  • navigation, re-layout navigation/ menu in lower resolution is very important
  • tools, don't reinvent the wheel. use bootstrap or similar libraries for responsive website

ref: responsive tutorial, Read best practices

function LotteryPicker(ticketMap) {
    this.ticketMap = ticketMap;
    this.generateNumber = Math.random;
}

LotteryPicker.prototype.pick = function() {
    var participants = Object.keys(this.ticketMap), 
        len = participants.length,
        total = 0;
   
    for(var i = 0; i<len; i++){
      total += this.ticketMap[participants[i]];
    }
    
    var raminingNumber = this.generateNumber()*total;
 
    
    for(var j =0; j<len; j++){
